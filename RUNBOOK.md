# RUNBOOK.md — Запуск стадий в Claude Code (блоки-промпты)

> Как давать Claude Code работу: **один блок = одна стадия**. Внутри блока агент
> работает автономно (цепочка субагентов), человек включается ТОЛЬКО на гейте.
> Между стадиями — приёмка Антона (PLAN.md). Не склеивать блоки в один запрос:
> это нарушает инвариант 6 CLAUDE.md (стадийная сдача, не гонка вперёд).
>
> Перед каждым блоком стек уже зафиксирован: **Astro + SQLite, SSR-эндпоинты в
> `web/src/pages/api/`**. Каталог `api/` (FastAPI) удалён на T0-0.

---

## Общая шапка (входит в начало каждого блока)

```
Контекст: прочитай CLAUDE.md, PROJECT_GUIDE.md, SPEC.md, PLAN.md,
PROJECT_STRUCTURE.md, DECISIONS.md, WORKFLOW.md, TASKS.md,
design/tokens.json, design/voice.md, design/references.md.
Действуй строго по инвариантам CLAUDE.md.

Запреты (делает человек, НЕ ты — CLAUDE.md §2): создание аккаунтов/паролей,
.env и секреты, права доступа, необратимое удаление данных, настройка оплаты,
деплой/слияние в main. Эти шаги опиши и отдай мне.

Границы: пиши только в свои каталоги (WORKFLOW §3). Цвета/типографику бери из
design/tokens.json, не хардкодь. Формы — нативный POST без JS; JS только как
улучшение поверх рабочей формы.

TDD: на каждую задачу сперва RED-тест (Vitest), потом GREEN. Зелёный — реальный,
не подогнанный.
```

## Правило перехода к V2
Не трогать V2 (самообслуживаемая админка, оплата, кабинеты, SEO-глубина), пока
V1 не принял реальные заявки на аренду и не пройден порог O5 (число постоянных
арендаторов — задаёт Антон). Без числа Gate-V2 не считается пройденным.
