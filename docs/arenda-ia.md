## 1. Карта блоков сверху вниз

Порядок из SPEC §2 (не меняем — выстроен под «решение по фото → условиям → доверию → действию»).

| № | Блок | Якорь | Роль в убеждении регулярности |
|---|------|-------|-------------------------------|
| 1 | **Hero** | `#hero` | Обещание «сделай это место своим», крупное фото лучшего зала, ОДИН первичный CTA. Не «аренда площади», а «регулярный слот = твоё место». |
| 2 | **Три зала — карточки** | `#zaly` | Специалист решает по фото своего будущего кабинета (A1). Карточка: фото, м², вместимость, оснащение, ставка с акцентом на регулярную (месяц/абонемент), почасовая вторично. CTA карточки → форма с предзаполнением зала. |
| 3 | **Почему здесь** | `#pochemu` | Атмосфера как польза ЕГО клиенту каждую неделю (voice.md: дерево, свет, чай, тишина, забота + «пять опор»). |
| 4 | **Окно доступности** | `#dostupnost` | Какие регулярные слоты свободны. Честная срочность («слотов конечное число»), не акции. **Данных O2 нет → заглушка.** |
| 5 | **Резиденты** | `#rezidenty` | Доверие: постоянство продаётся через постоянство других. **Данных O3 нет → заглушка.** |
| 6 | **Форма заявки** | `#zayavka` | Главная конверсия. Полная форма (§4.1), «регулярная» по умолчанию. |
| 7 | **FAQ аренды** | `#faq` | Снимает возражения: как оформляется, что входит, депозит/правила, можно ли разово. |
| — | **Повтор CTA (низ)** | → `#zayavka` | Дублирует первичный CTA после FAQ (SPEC §2). |

Логика: желание (фото зала) → обоснование (атмосфера = польза его клиенту) → дефицит (слоты) →
социальное доказательство (резиденты) → действие (форма) → дожатие (FAQ).

## 2. Выше линии сгиба

Первичный CTA — ОДИН: **«Закрепить за собой слот»** (формулировку финализирует copywriter) → `#zayavka`.

**Мобайл:** Экран 1 — крупное фото лучшего зала + заголовок-обещание + 1 строка подзаголовка +
ОДИН CTA. Экран 2 — первая карточка зала «наезжает» снизу (сигнал листать). Sticky-CTA — только
как прогрессивное улучшение, не критичный путь.

**Десктоп:** Экран 1 — фото лучшего зала крупно, поверх/справа заголовок-обещание + подзаголовок +
ОДИН CTA + микро-якорь «3 зала · регулярные слоты». В зоне hero только одна акцентная кнопка.

Правило: выше сгиба — ровно одна кнопка-действие.

## 3. Пользовательский поток

```
Вход (VK/Telegram/сарафан/прямая ссылка)
  → [1] HERO (CTA «Закрепить за собой слот») ─┐
  → [2] ТРИ ЗАЛА (CTA «Хочу вести здесь» → предзаполняет hall) ─┤
  → [3] ПОЧЕМУ ЗДЕСЬ ─┤
  → [4] ОКНО ДОСТУПНОСТИ (или заглушка) ─┤
  → [5] РЕЗИДЕНТЫ (или заглушка) ─┤
  → [6] ФОРМА ЗАЯВКИ ◄─ (все CTA сходятся сюда), нативный POST
  → [7] FAQ
  → [повтор CTA] → #zayavka
  → СТРАНИЦА «СПАСИБО» (подтверждение + следующий шаг)
```

Все «действия» ведут в одну форму. Параллельной конверсии (заявка на визит) на лендинге НЕТ —
это другой сегмент и другая страница.

## 4. Состояния

### Форма заявки (`#zayavka`)
- **Пусто:** поля пустые/с дефолтами; формат = «регулярная» предвыбран; зал предзаполнен если пришёл
  с карточки; согласие снято; подсказки видны.
- **Заполнение:** с JS — мягкая валидация на лету; без JS — обычный ввод, проверка на сервере.
- **Ошибка валидации:** без JS сервер возвращает ту же форму с сохранёнными значениями (кроме согласия),
  сообщения у полей + сводка вверху с якорями; фокус на первое ошибочное поле; `idempotency_key` тот же
  (повтор не плодит лид). С JS — то же без перезагрузки.
- **Успех:** POST→redirect→GET на страницу «спасибо»: подтверждение + понятный следующий шаг (кто и в
  какой срок свяжется, по какому каналу). Цель Метрики `arenda_lead` фиксируется здесь.

### «Окно доступности» (O2 нет) — честная заглушка
Не выдумываем слоты. «Свободные регулярные слоты уточняются — впишите желаемые дни и время в заявке,
подберём». Опц. статичная сетка «дни × утро/день/вечер» БЕЗ проставленной занятости. CTA → форма.
Никаких «осталось 2 слота».

### «Резиденты» (O3 нет) — честная заглушка
Не выдумываем людей/имена/фото. По умолчанию — приглашение «Станьте одним из первых, кто закрепит за
собой слот». Без фейковых аватарок с именами; допустимы нейтральные силуэты как заведомо-пустые места.
Верстка должна безболезненно наполниться реальными карточками, когда O3 закроется.

**Открытый вопрос (Антон):** скрывать пустые блоки O2/O3 или показывать заглушку-приглашение?
Рекомендация — приглашение (работает на конверсию).

## 5. Поля формы заявки (SPEC §4.1)

Видимые (порядок от простого к сложному, согласие в конце):

| Поле | Тип | Обяз. | Подпись (черновик) | Подсказка |
|------|-----|:----:|---------------------|-----------|
| Имя | text | да | «Как вас зовут» | — |
| Контакт | tel/text | да | «Телефон или мессенджер» | «Телефон, Telegram, WhatsApp» |
| Вид деятельности | select+«другое» | да | «Чем занимаетесь» | массаж · психолог · коворкинг · телесные практики · другое |
| Желаемый зал | radio/select | нет | «Какой зал» | Большой 70 · Малый 30 · Чайный 45 · «пока не выбрал»; предзаполняется из карточки |
| Формат | radio | да | «Формат аренды» | «Регулярная» (предвыбрано) · «Разовая» |
| Регулярность | text/textarea | условно* | «Какие дни и время» | «Например: вт и чт, 18:00–21:00». *Обязательно при «регулярная» |
| Желаемый старт | text/month | нет | «Когда хотели бы начать» | «Месяц или примерная дата» |
| Комментарий | textarea | нет | «Что важно учесть» | «Оборудование, число клиентов, вопрос» |
| Согласие | checkbox | да | «Согласен(а) на обработку данных» + ссылка **/privacy** | без галочки не отправляется (серверная проверка тоже) |

Служебные поля (CLAUDE.md §1, SPEC §4):

| Поле | Тип | Назначение |
|------|-----|------------|
| `idempotency_key` | hidden | UUID, сгенерён при РЕНДЕРЕ (до отправки). Повтор = тот же лид. Не из ответа внешней системы. = `lead.id`. |
| honeypot | скрытое поле-ловушка | антиспам без CAPTCHA; `aria-hidden`, вне табордера. |
| время-на-форме | hidden timestamp/токен | антиспам: слишком быстрая отправка = бот. |
| `source_page` | hidden | = `/arenda` (`lead.source_page`). |
| `cta_origin` | hidden (опц.) | какой CTA/карточка привёл. Не обязателен в M0. |

Доступность формы (WCAG AA): `<label for>` у каждого поля; обязательность текстом, не только цветом;
ошибки через `aria-describedby`; фокус на первую ошибку; honeypot вне доступа клавиатуры/скринридера;
вся форма с клавиатуры; работает без JS.

## 6. Иерархия CTA и якоря

Один первичный CTA: «Закрепить за собой слот» → `#zayavka`.
1. **Первичный** — Hero (единственный акцент выше сгиба).
2. **Контекстные** — «Хочу вести здесь» в каждой карточке (+предзаполнение зала).
3. **Поддерживающие** — мягкие CTA в «Почему здесь»/«Доступность»/«Резиденты» → форма.
4. **Повтор внизу** — после FAQ.

Якоря: `#hero` · `#zaly` · `#pochemu` · `#dostupnost` · `#rezidenty` · `#zayavka` · `#faq`.
Второй конкурирующей конверсии на лендинге нет.
