# PLAN — Стадийный план (TDD, RED→GREEN)

> Дизайн-first. Каждый шаг: тест RED → код GREEN → приёмка Антоном → следующий шаг.
> Никакой автономной гонки вперёд. Стадии маленькие и проверяемые.

---

## M0 — Каркас и дизайн-система (вертикальный срез)

Цель: увидеть «красивую картинку» и доказать, что бренд-бук переносится в код.

1. Репо-скелет по PROJECT_STRUCTURE; стек зафиксирован (Astro+SQLite по умолч.).
2. `design/tokens.json` → CSS-переменные; self-hosted шрифты (Raleway/Montserrat/
   Open Sans, статичные TTF, кириллица). **Тест:** контраст-гейты из tokens
   проходят (оранжевый не используется как текст; кнопки #D35400+белый ≥16px).
3. Базовый layout + типошкала + компоненты (кнопка, карточка, форма) на токенах.
4. **Лендинг аренды `/arenda` на плейсхолдер-фото** — герой проекта целиком,
   статикой. Это и есть «картинка», которую показываем и критикуем.

**Gate M0:** Антон принимает визуал лендинга аренды. До приёмки дальше не идём.

## M1 — Главная конверсия: заявка на постоянную аренду (сквозной путь)

1. Форма аренды **без JS** → POST-эндпоинт. **Тест RED:** POST создаёт ровно один
   `lead(type=arenda)`; повтор того же `idempotency_key` — не создаёт второй.
2. Страница «спасибо» + следующий шаг.
3. Метрика-цель `arenda_lead`.
4. Прогрессивное улучшение: JS-валидация поверх рабочей нативной формы.

**Gate M1:** реальная тестовая заявка проходит и видна; дедуп подтверждён.

## M2 — Контент и расписание

> ⛔ **Блокер O1 (см. DECISIONS.md):** правило приоритета вечернего слота (своё
> занятие vs аренда) не решено. До решения расписание публикуется только
> без-конфликтными слотами (`conflict_flag` держит пересечения скрытыми). Решение —
> на Антоне, до завершения M2.

1. Модель данных + сидер; админ-CRUD (одна роль) для залов/специалистов/слотов.
2. Страница `/raspisanie` с моделью «два CTA на слот»; флаг конфликта вечерних.
3. Контентные страницы (направления, преподаватели, чайная, туры, психолог) из
   `content_page`; голос бренда (copywriter), проверка brand-guardian.
4. Форма визита + регистрация на события (тот же паттерн идемпотентности).

**Gate M2:** контент залит, расписание публикуется без пересечений.

## M3 — Полировка и запуск

1. Реальные фото залов (от фотографа) заменяют плейсхолдеры; brand-guardian-ревью.
2. Schema.org LocalBusiness; перенос /privacy; редиректы со старых URL Tilda.
3. QA: формы, мобильная версия, без-JS, доступность-минимум, скорость.
4. Метрика проверена на всех целях.

**Gate M3:** прод-чеклист зелёный → публикация.
